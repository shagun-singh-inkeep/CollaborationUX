# Ways to improve UX for collaboration



## Timeline Changes

- **No "changes since I last looked."** The Timeline is a full polled list with no session high-water mark / unread state — a returning user can't tell what's new at a glance.
- there's no workspace-wide "everything that changed recently" view.
- *add* an incremental per-entry diff (entry vs its parent) for the "what did this edit do" feed — which is what the rendered-block-diff / Notion-style updates feature really wants. They're not the same feature; today OK only has the first.

## Permissioning

- **Comments / annotations / suggestions** — entirely absent.
- **View-only / permissions** — no role model; access == GitHub repo access, not enforced inside OK.
