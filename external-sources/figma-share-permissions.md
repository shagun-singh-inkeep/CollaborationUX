# Figma — share files & prototypes (source)

> Raw reference captured from Figma Help (<https://help.figma.com/hc/en-us/articles/360040531773-Share-files-and-prototypes>) on 2026-07-06. Preserved to ground [[guides/view-only-access|the view-only access research]].

## Access tiers

- **Can view** — see the file, cannot change it. (Binary view/edit model.)
- **Can edit** — modify file content.

## What a viewer can still do

- **Inspect** designs and (on the presence side) still appears in the file — Figma viewers remain live participants; the [[guides/figma|Figma study]] notes presence is the product, and a viewer is still visible.
- **Copy / share / export** is an *advanced* sharing toggle ("Allow viewers to copy, share, and export") on paid plans — normally **off** for viewers by default (parallels Google's download restriction).

## Sharing methods

- **Link sharing** — copy a direct link to file or prototype, no invite required.
- **Direct invitation** — invite individuals with a per-person role.
- **Org/workspace** — "Anyone in [Org/Workspace]" on paid plans, uniform role.
- **Prototype-only link** — present designs to clients/stakeholders without granting access to the full design file. (On Starter, presentation-view viewers can still open the source file — a leakage caveat.)
